# 测试设计文档：Epic 1 & 2 集成测试与 E2E 测试

## 📋 Executive Summary

本文档定义了 **Epic 1（搜索与数据获取）和 Epic 2（结果展示与交互）** 的完整测试策略，包括：

- **风险评估矩阵** — 识别 4 个高风险项（Risk ≥ 6）
- **测试覆盖计划** — 24 个测试场景，分为 P0/P1/P2/P3 优先级
- **执行策略** — PR/Nightly/Weekly 三层执行模式
- **资源估计** — 总计 ~60-90 小时
- **质量门槛** — P0 100% 通过，P1 ≥95% 通过

## 🚨 风险评估矩阵

### **高风险项目 (Risk Score ≥ 6)**

#### 1️⃣ **Gemini API 超时与重试失败** [TECH]
- **概率:** 3 (High) — API 网络不稳定，30s 超时容易触发
- **影响:** 3 (High) — 搜索中断，用户无法获得结果
- **风险分数:** 9 🔴
- **缓解方案:**
  - ✅ 单元测试：验证 3 次重试逻辑 (2s, 4s, 8s 指数退避)
  - ✅ 集成测试：Mock timeout → verify retry → verify success
  - ✅ E2E 测试：模拟网络延迟 → verify 超时处理
- **测试用例:** `P0-2` (Gemini API 超时重试)

#### 2️⃣ **API 密钥泄露** [SEC]
- **概率:** 2 (Medium) — 日志输出、错误消息可能暴露
- **影响:** 3 (High) — 安全漏洞，Gemini 账户被滥用
- **风险分数:** 6 🟠
- **缓解方案:**
  - ✅ 单元测试：验证日志中不出现完整 API 密钥
  - ✅ 集成测试：审计所有日志输出，确保仅显示前缀 "sk_...***"
  - ✅ 代码审查：检查错误消息中是否包含密钥
- **测试用例:** `P1-2` (API 密钥缺失处理)

#### 3️⃣ **数据库并发冲突** [DATA]
- **概率:** 2 (Medium) — 多用户并发搜索时触发
- **影响:** 3 (High) — 数据不一致、重复记录、数据丢失
- **风险分数:** 6 🟠
- **缓解方案:**
  - ✅ 集成测试：3 个并发搜索 → 验证无死锁、数据一致
  - ✅ 数据库测试：duplicate_count 准确性验证
  - ✅ 事务隔离级别检查
- **测试用例:** `P1-4` (并发搜索场景)

#### 4️⃣ **CSV 导出超时 (大数据集)** [PERF]
- **概率:** 2 (Medium) — 500+ 条记录导出时触发
- **影响:** 2 (Medium) — 用户导出失败，需重试
- **风险分数:** 4 🟡
- **缓解方案:**
  - ✅ 性能测试：5MB 数据集导出 < 2 秒
  - ✅ E2E 测试：大数据导出 → 文件完整性验证
  - ✅ 流式导出优化（如需）
- **测试用例:** `P1-6` (大文件导出)

## 📊 测试覆盖矩阵

### **P0 — 关键路径 (8 个用例) ⭐ 必须 100% 通过**

| ID | 测试场景 | 测试级别 | 故事 | 关键验证点 |
|----|---------|---------|------|-----------|
| P0-1 | Gemini API 成功调用 | 集成 | 1-3,1-4 | 搜索请求 → API 响应 → 数据完整 |
| P0-2 | Gemini API 超时重试 | 集成 | 1-3 | 第 1 次超时 → 等待 2s → 重试 → 最终成功/失败 |
| P0-3 | 数据解析与存储 | 集成 | 1-5 | JSON/CSV 响应 → 解析字段正确 → SQLite 存储 → 验证完整性 |
| P0-4 | 结果分页 API | API | 2-1 | GET /api/search/results?page=1,per_page=20 → 返回 20 条 + 元数据 |
| P0-5 | 卡片视图渲染 | E2E | 2-2 | 搜索 → 点击搜索按钮 → 卡片视图显示 → 所有 23 字段可见 |
| P0-6 | 表格视图渲染 | E2E | 2-3 | 搜索 → 点击表格视图 → 显示表格 → 排序功能正常 |
| P0-7 | 视图切换与状态持久化 | E2E | 2-4 | 卡片 → 表格 → 页面刷新 → 表格视图恢复 + 排序/分页状态保留 |
| P0-8 | CSV 导出基础功能 | E2E | 2-5 | 导出按钮 → 选择范围 → 生成文件 → 23 列正确 + UTF-8 BOM + CRLF |

### **P1 — 重要边界 (8 个用例) 需要 ≥95% 通过**

| ID | 测试场景 | 测试级别 | 描述 | 关键验证点 |
|----|---------|---------|------|-----------|
| P1-1 | 无搜索结果处理 | API | 空结果集返回 | 200 + `{companies: []}` + 正确的 total_count = 0 |
| P1-2 | API 密钥缺失 | 单元 | 启动时 GEMINI_API_KEY 为空 | 清晰错误消息 + 日志不包含敏感信息 |
| P1-3 | 大数据集分页 | 集成 | 500+ 条记录分页 | 分页偏移正确 + 性能无显著下降 |
| P1-4 | 并发搜索 | 集成 | 3 个并发搜索同时执行 | 数据不混乱 + 无死锁 + duplicate_count 准确 |
| P1-5 | localStorage 损坏恢复 | 前端 | 清除 localStorage 后刷新 | 应用恢复默认状态（卡片视图、第 1 页） |
| P1-6 | 大文件导出性能 | E2E | 导出 5MB 数据 | 耗时 < 2 秒 + 文件完整 + 未截断 |
| P1-7 | 网络请求超时 | E2E | 模拟 10s+ 延迟 | 显示超时消息 + 提供重试按钮 + 重试成功 |
| P1-8 | API 响应格式验证 | API | 验证所有 API 响应 | 信封格式正确 + 所有必需字段存在 + 数据类型匹配 |

### **P2 — 边界情况 (5 个用例) 需要 ≥80% 通过**

| ID | 测试场景 | 测试级别 | 描述 |
|----|---------|---------|------|
| P2-1 | CSV 特殊字符处理 | 单元 | 逗号、引号、换行、Unicode → 正确转义 + 导入 Excel 不乱码 |
| P2-2 | 重复数据检测 | 单元 | duplicate_count 字段准确 + 去重算法正确 |
| P2-3 | 键盘导航 | E2E | Tab/Enter/Arrow 键 → 视图切换 + 表格行选中 + 导出对话框操作 |
| P2-4 | 移动响应式 | E2E | 768px 以下屏幕 → 卡片为默认 → 功能完整 → 无布局破裂 |
| P2-5 | 排序状态持久化 | E2E | 表格排序 → 刷新 → 排序保留 + 分页位置保留 |

### **P3 — 性能与可选项 (3 个用例) 可选但推荐**

| ID | 测试场景 | 描述 |
|----|---------|------|
| P3-1 | 性能基准 | 单个搜索 < 5s; 100 条渲染 < 500ms; CSV 生成 < 2s |
| P3-2 | 内存泄漏检测 | 10 次搜索循环 → 内存稳定（增长 < 5%） |
| P3-3 | 可访问性 (A11y) | WCAG 2.1 AA 合规 + 屏幕阅读器兼容 |

## 🔄 执行策略

### **三层执行模式**

```
每个 PR 提交
  ↓

  ├─ P0 全套 (8 个)
  └─ P1 快速子集 (4 个关键路径)

  PASS → 允许合并

  ↓
[夜间完整套件] ~45-60 分钟
  ├─ P0 全套 (8 个)
  ├─ P1 全套 (8 个)
  └─ P2 全套 (5 个)
  ↓
  报告覆盖率、性能指标

每周日 2 AM
  ↓
[周末压力测试] 2-3 小时
  ├─ 并发测试 (P1-4)
  ├─ 大数据测试 (P1-6)
  ├─ 性能基准 (P3-1)
  └─ 内存泄漏 (P3-2)
  ↓
  长期稳定性报告
```
