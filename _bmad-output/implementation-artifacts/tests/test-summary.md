## 🧪 测试详情

### 1️⃣ 完整搜索管道测试（14个）

#### 核心功能测试
- ✅ **JSON 响应处理** — 验证 Gemini JSON 响应的完整处理流程
  - 搜索表单 → API 调用 → 文件保存 → 格式验证 → 数据解析 → 数据库写入
  - 测试通过：新增公司 ≥ 0，搜索历史记录已创建

- ✅ **CSV 响应处理** — 验证 CSV 格式响应的解析与入库
  - 正确处理逗号分隔的数据
  - 数据库一致性检查

- ✅ **Markdown 响应处理** — 验证表格格式的解析
  - Markdown 表格 → 数据提取 → 数据库插入

#### 关键业务逻辑测试
- ✅ **去重逻辑** — 跨多次搜索验证 LinkedIn URL 去重
  - 第一次搜索：新增 N 家公司
  - 第二次搜索：相同数据 → 检测到重复 → 跳过插入

- ✅ **统计计算** — 验证平均评分和高优先级计数
  - 新增公司的平均评分（avg_score）正确计算
  - 高优先级公司计数（score ≥ 8）准确

- ✅ **评分归一化** — 验证评分值边界处理
  - 无效评分（< 1 或 > 10）被正确约束
  - 非数字评分处理为 0 或默认值

- ✅ **LinkedIn URL 规范化** — 验证 URL 标准化逻辑
  - 去除 `https://`、`www.` 前缀
  - 去除末尾斜杠
  - 转换为小写
  - 结果用于去重

- ✅ **缺失字段处理** — 验证不完整数据的容错性
  - 缺失可选字段用默认值填充
  - 处理继续，不中断管道

#### 数据完整性测试
- ✅ **搜索历史准确性** — 验证统计数据准确性
  - new_count、duplicate_count 一致性
  - avg_score 计算正确

- ✅ **大数据集处理** — 50 家公司批量处理
  - 性能正常（< 5 秒）
  - 数据完整性保持

- ✅ **并发搜索隔离** — 不同国家/关键词搜索独立执行
  - Vietnam 搜索不污染 Thailand 搜索
  - 统计数据分别正确计算

- ✅ **数据一致性** — 整个管道中的数据完整性验证
  - 必填字段（name, country）不为 None
  - 邮箱格式验证（包含 @）
  - 评分范围 0-10 或 None

#### 边界情况处理
- ✅ **空响应处理** — Gemini 返回空列表的处理
  - 管道正常完成，new_count = 0

- ✅ **去重追踪** — 搜索历史中的去重计数记录
  - 第二次搜索正确记录重复数
